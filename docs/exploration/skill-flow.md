# AI-First Skill Flow

## 目标分层

### L1：页面级首轮闭环

回答的问题：

- 页面能否稳定进入
- 页面显式暴露了哪些能力
- 页面级 workflow 是否已形成
- 页面级 open questions 是否已归零

### L2：业务流闭环

回答的问题：

- 页面之间如何组成关键任务链路
- 哪些 workflow 已经跨页闭环
- 哪些页面只是“可达”，还没有进入业务流判断

### L3：状态空间闭环

回答的问题：

- 正常态之外还覆盖了哪些状态
- 哪些边界态仍是盲区
- 哪些异常行为是产品规则，哪些只是探针噪音

## 角色分工

### Playwright

Playwright 只做 4 件事：

- 进入页面
- 执行动作
- 采集证据
- 在失败时做有限自愈

它不是最终分析者，不负责判定 L2/L3 是否完成。

### AI

AI 负责 4 件事：

- 从探针证据中识别功能候选
- 将页面能力组织成业务流理解
- 将页面行为组织成状态空间理解
- 给出开放问题与下一轮探针建议

### Controller

控制器负责串起两者：

- 触发探针
- 收集页面级证据包
- 检查页面级 open questions
- 决定是否继续下一轮

控制器主要服务于 L1，不应直接代表最终探索完成判断。

## 执行顺序

### 阶段 1：整站首次摸底

目标：建立 L1 页面地图。

输出：

- 页面级 metadata
- 页面索引
- 页面专属 workflow 初稿

### 阶段 2：定向页面闭环

目标：把重点页面压到 L1 收敛。

输出：

- 更稳定的页面 workflow
- 更少的 open questions
- 更明确的边界态判断

### 阶段 3：业务流梳理

目标：从页面 workflow 提升到 L2 业务流理解。

输出应该回答：

- 哪些页面属于同一任务链路
- 起点、转折点、终点态分别是什么
- 哪条链路仍然没有真正闭环

### 阶段 4：状态空间梳理

目标：从页面成功提升到 L3 状态覆盖。

输出应该回答：

- 哪些状态已经覆盖
- 哪些状态仍缺证据
- 哪些状态只应算边界态验证，不应误当失败

### 阶段 5：整站刷新

目标：把 latest 结果恢复成整站视图。

规则：

- 任何局部页面 run 之后，如要更新仓库级最新视图，必须做整站刷新。

## 产物映射

### L1 主要产物

- `probeEvidence`
- `featureFindings`
- `aiAnalysis`
- `iterationLogs`
- `page-details-index.md`

### L2 应回答但当前未系统化产出的内容

- 关键业务流清单
- 每条业务流的起点页 / 终点态 / 成功判定
- 哪些页面只是孤立页面结论

### L3 应回答但当前未系统化产出的内容

- 状态覆盖矩阵
- 边界态与异常态清单
- 未覆盖状态的风险说明

## 当前最重要的约束

1. 页面级收敛不等于最终探索完成。
2. 优先补证据，再补结论。
3. 局部 run 之后必须警惕 latest 结果被误读成整站结论。
4. 状态空间缺失时，不能只因为页面 workflow 为 PASS 就提前结束。
